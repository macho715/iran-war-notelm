# 작업 이력 (2026-03-05)

## 프로젝트 개요

이란-UAE 분쟁 실시간 모니터링 시스템.
GHA에서 30분마다 뉴스를 수집·분석해 Telegram으로 경보를 전송하고,
결과를 urgentdash React 대시보드(Vercel)에 표시한다.

---

## 최종 시스템 구조

```
GHA monitor.yml (30분마다 자동 실행: :07, :37)
  │
  ├─ 1) 병렬 스크래핑
  │     UAE 미디어(Playwright) + SNS(Playwright) + RSS(httpx)
  │
  ├─ 2) HyIE 상태 갱신
  │     tier0/1/2 신호 + 기사 신호 → state/hyie_state.json
  │
  ├─ 3) 중복 제거 (run-local hash + DB canonical_url)
  │
  ├─ 4) 신규 기사 있을 때만:
  │     a) NotebookLM 업로드
  │     b) Phase2 AI 분석 (query → JSON) / rule-based fallback
  │     c) 교차검증 게이트 (HIGH/CRITICAL: 2소스 + 2키워드)
  │     d) 즉시 경보 → Telegram
  │     e) 보고서 + Telegram 전송
  │
  ├─ 5) Postgres/SQLite 저장 (runs, articles, outbox)
  │
  └─ 6) urgentdash-live 브랜치에 hyie_state.json publish
        → live/hyie_state.json → Vercel React 앱이 읽음

urgentdash React 대시보드 (Vercel)
  - fast poll 30초: 로컬 API → GitHub raw URL fallback
  - full sync 30분: GitHub raw hyie_state.json
  - 표시: 위협 맵, 타임라인, 경로, 체크리스트, AI 분석 결과
```

### 5. STALE 19시간 문제 해결 (핵심)

**근본 원인**: `urgentdash-live-publish.yml` 삭제 → `live/hyie_state.json`이 GitHub에 업데이트 안 됨

**해결**: `monitor.yml`에 publish 스텝 통합

```yaml
jobs:
  run-monitor:
    permissions:
      contents: write   # ← 필수 (git push 권한)
    steps:
      ...
      - name: Publish hyie_state.json to urgentdash-live branch
        if: always()
        run: |
          cp state/hyie_state.json /tmp/hyie_state.json
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git fetch origin urgentdash-live
          git checkout urgentdash-live
          mkdir -p live
          cp /tmp/hyie_state.json live/hyie_state.json
          git add live/hyie_state.json
          git diff --cached --quiet && echo "No changes" || \
            git commit -m "chore: update hyie_state.json [skip ci]" && \
            git push origin urgentdash-live
```

- `[skip ci]` 태그 → Vercel 재배포 트리거 방지
