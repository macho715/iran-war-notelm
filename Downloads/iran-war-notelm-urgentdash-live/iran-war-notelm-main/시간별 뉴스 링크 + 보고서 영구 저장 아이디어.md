## 시간별 뉴스 링크 + 보고서 영구 저장 아이디어

현재 상태: 매시간 수집된 기사와 분석 결과가 Telegram/WhatsApp으로만 전송됨. NotebookLM에 소스는 누적되지만, 구조화된 검색/조회가 어려움.

---

### Option A: 로컬 JSON 아카이브 (가장 빠른 구현)

```
reports/
  2026-03-01/
    14-00.json   ← {articles: [...], analysis: {...}, notebook_url: "..."}
    15-00.json
    16-00.json
  2026-03-02/
    ...
```

- `hourly_job` 완료 후 `reports/{date}/{time}.json`에 기사 목록 + AnalysisResult + notebook_url 저장
- 장점: 의존성 0, 즉시 구현 가능, `jq`로 검색 가능
- 단점: 로컬 PC 한정, 팀 공유 불가

---

### Option B: SQLite (검색 가능한 로컬 DB)

```
iran_monitor.db
  ├── articles (id, timestamp, source, title, link, threat_level, city)
  └── reports  (id, timestamp, threat_level, score, sentiment, summary, notebook_url)
```

- 장점: SQL로 "지난 7일 HIGH 이상 기사만" 같은 쿼리 가능, 트렌드 분석 기반
- 단점: 여전히 로컬

---

### Option C: NotebookLM MCP 쿼리 아카이브

현재 NotebookLM에 매시간 텍스트 소스가 쌓이고 있음. 이걸 활용하는 방법:

1. **소스 업로드** (현재 동작) — 기사 원문이 NotebookLM에 계속 누적
2. **쿼리 결과를 별도 저장** — `analyze_with_notebooklm_client` 응답(JSON)을 로컬/클라우드에 아카이빙
3. **NotebookLM을 "과거 맥락 DB"로 활용** — 다음 사이클에서 "지난 24시간 트렌드는?" 같은 쿼리를 날려 시계열 분석

```
매시간:
  기사 수집 → NotebookLM 소스 추가 (누적)
             → NotebookLM 쿼리 (현재 위협 + 과거 대비 변화)
             → 쿼리 결과 JSON → 로컬/Notion/Sheets에 저장
```

- 장점: NotebookLM이 자체적으로 맥락을 축적, 시간이 갈수록 분석 품질 향상
- 단점: 소스가 너무 많아지면 정리 필요 (quota), VPN 의존

---

### Option D: Google Sheets (팀 공유 + 실시간)

```
[Iran-UAE Monitor Log]
| 시간 | 위협 | 점수 | 감성 | AD | DXB | 기사1 | 기사2 | 기사3 | NotebookLM |
|------|------|------|------|-----|-----|-------|-------|-------|------------|
| 03-01 14:00 | HIGH | 78 | 긴급 | HIGH | MED | link1 | link2 | link3 | nb-url |
| 03-01 15:00 | LOW  | 22 | 일반 | LOW  | LOW | link4 | ...   |       | nb-url |
```

- `gspread` + Google Sheets API로 매시간 행 추가
- 장점: 팀원 누구나 브라우저로 실시간 확인, 필터/차트 가능, 무료
- 단점: Google API 인증 필요, 시트 크기 제한 (충분함)

---

### Option E: Notion 데이터베이스 (UPGRADE_GRAND_PLAN Phase 3에 이미 있음)

```
[Notion DB: Iran-UAE Monitor]
  Properties: 시간, 위협레벨, 점수, 감성, 아부다비, 두바이, 요약, 기사링크들, NotebookLM URL
  Views:
    - 테이블: 전체 로그
    - 갤러리: HIGH/CRITICAL만 필터
    - 캘린더: 일자별 위협 추이
```

- `notion-client` Python SDK로 매시간 페이지 생성
- 장점: 리치 포맷, 팀 공유, 필터/뷰, 모바일 앱
- 단점: Notion API 키 필요

---

### Option F: GitHub 자동 커밋 아카이브

```
매시간 hourly_job 완료 후:
  reports/latest.md 업데이트 + git commit + push
  → GitHub repo가 곧 히스토리 DB
  → git log로 시계열 조회
```

- 장점: 버전 관리 자동, GitHub Pages로 공개 가능
- 단점: 매시간 커밋 노이즈

## 추천 조합

| 우선순위 | 조합 | 이유 |
|----------|------|------|
| **즉시** | A (JSON) + C (NotebookLM 누적) | 의존성 0, 10줄 추가로 구현 |
| **1주 내** | B (SQLite) + D (Google Sheets) | 검색 + 팀 공유 |
| **Phase 3** | E (Notion) | 리치 UI, 장기 아카이빙 |

핵심 아이디어: **NotebookLM은 "분석 엔진"으로 계속 쓰고, 결과물(기사 링크 + 보고서)은 별도 저장소에 구조화해서 저장**하는 것이 가장 실용적입니다. NotebookLM 자체는 아카이브 DB가 아니라 AI 분석 도구이므로, 출력물을 꺼내서 보관하는 레이어를 추가하는 것이 맞습니다.

구현하고 싶은 옵션을 알려 주시면 Agent 모드에서 바로 작업하겠습니다.


## 인터넷 조사 기반 — 시간별 뉴스/보고서 저장 아이디어

### 1. `notebooklm-py` 라이브러리로 전체 아티팩트 다운로드

[teng-lin/notebooklm-py](https://github.com/teng-lin/notebooklm-py) (v0.3.2)가 현재 가장 강력한 옵션:

- 생성된 모든 아티팩트를 로컬로 다운로드: **MP3(팟캐스트), PDF, PNG(인포그래픽), CSV, JSON, Markdown**
- 쿼리 결과 + 대화 히스토리 + 소스 메타데이터 모두 추출 가능
- Python 3.10~3.14 호환

```
매시간:
  기사 업로드 → NotebookLM 쿼리 → 분석 결과
  → notebooklm-py로 대화 히스토리 + 소스 목록 JSON 다운로드
  → reports/{date}/{time}/ 폴더에 저장
```

---

### 2. Apify NotebookLM API (무료 베타)

[Apify NotebookLM API Actor](https://apify.com/clearpath/notebooklm-api) — 현재 **무료 베타**:

- 전체 대화 히스토리 (Q&A 스레드)
- 소스 메타데이터 (제목, URL)
- **Citation mapping** — 어떤 소스가 어떤 응답에 인용됐는지
- 출력 형식: JSON, CSV, Markdown, Excel
- Audio/Video Overview, 슬라이드, 인포그래픽, 플래시카드까지 추출

```python
# Apify 클라이언트 (pip install apify-client)
from apify_client import ApifyClient
client = ApifyClient("YOUR_TOKEN")
run = client.actor("clearpath/notebooklm-api").call(
    run_input={"notebookUrl": notebook_url, "outputFormat": "json"}
)
items = client.dataset(run["defaultDatasetId"]).list_items().items
# → 전체 대화+소스+인용 JSON
```

### 3. RSS + AI 분석 → Notion 자동 저장 (실제 사례)

[Smax2k/news-py](https://github.com/smax2k/news-py) — 실제 운영 중인 패턴:

```
RSS 수집 → ChatGPT 분석 → Notion DB에 자동 저장
```

이걸 현재 파이프라인에 적용하면:

```
UAE 스크랩 → NotebookLM 분석 → Notion DB에 행 추가
  Properties: 시간, 위협레벨, 점수, 감성, 기사링크들, 분석요약, NotebookLM URL
```

Notion은 **캘린더 뷰, 필터, 모바일 앱**이 있어서 팀 공유에 최적.

| **RSS + feedparser** | 무료 | 가장 안정적, 구조화 |

### 5. 종합 추천: 3-레이어 아카이브

```
Layer 1: NotebookLM (분석 엔진 + 맥락 누적)
  ↓ notebooklm-py 또는 Apify로 추출
Layer 2: 로컬 SQLite (빠른 검색, 트렌드 쿼리)
  ↓ 동기화
Layer 3: Notion 또는 Google Sheets (팀 공유, 시각화)
```

**매시간 파이프라인 흐름:**

```
스크랩 → NotebookLM 업로드+쿼리
       → AnalysisResult JSON
       → (1) SQLite INSERT (articles + reports 테이블)
       → (2) Notion DB 행 추가 (또는 Google Sheets append)
       → (3) notebooklm-py로 대화히스토리 JSON 백업 (일 1회)
       → Telegram/WhatsApp 전송
```
